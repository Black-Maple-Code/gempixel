# Quick Task 260713-01y: Redesign Grid + Symbols Allocation (Distinct Glyphs, No Digits) Summary

Rebuilt `src/engine/symbols.ts` so the Grid + Symbols view leads with 105 easily-distinguishable monochrome shape-glyphs, pushes 19 unambiguous capital letters to a last resort, and contains no digits anywhere — eliminating O/0, I/1, Z/2, S/5, B/8 confusion and the ugly "glyph+digit" two-char combos on large palettes.

> Fix-forward applied (commit `db586e3`): an in-browser rendering + near-duplicate audit passed all 122 glyphs for tofu/emoji but flagged 17 as not easily distinguishable at cell size (pure rotations and tiny-interior twins). Those were pruned (glyph tier 122 -> 105); the tail now falls to the clean `SAFE_LETTER_SYMBOLS` capitals. `CURATED_SYMBOLS` = 105 glyphs + 19 letters = 124 (still > 118), all unique, zero digits.

## What Was Built

**Task 1 — symbols.ts rebuilt (symbols-first ordered pool):**
- `GLYPH_SYMBOLS` (105 glyphs after the distinctness prune) — the entire proven-safe production set kept verbatim as the bold lead (`♣ ♦ ♥ ♠ ▲ ▼ ◆ ● ■ ★ …`, 46 glyphs), then a vetted, deduped, emoji-free, distinctness-pruned expansion (corner triangles, half/quadrant circles, hatched/partitioned squares, star/florette dingbats, crosses, card-suit outlines, single + double arrows). All BMP, all text-presentation default. Pure rotations and tiny-interior twins removed so no two glyphs read alike at ~10px.
- `SAFE_LETTER_SYMBOLS` (19) — `A C D E F H J K L M N P R T U V W X Y`, excluding the digit/look-alike confusables B, G, I, O, Q, S, Z.
- `CURATED_SYMBOLS = [...GLYPH_SYMBOLS, ...SAFE_LETTER_SYMBOLS]` (124). `LETTER_SYMBOLS`/`NUMBER_SYMBOLS` removed; no digits.
- `generateSymbolAllocation` (frequency-desc + `localeCompare` tie-break), the `>poolSize` multi-char base+suffix overflow fallback, `symbolFontPx`, and `getContrastColor` preserved byte-for-byte in behavior — the B4 uniqueness net is intact, now only firing when a palette exceeds the whole enlarged pool.
- Top doc comment rewritten to describe the new glyphs-first / letters-last / no-digits order.

**Task 2 — symbols.test.ts rewritten invariants:**
- Imports updated to `GLYPH_SYMBOLS` / `SAFE_LETTER_SYMBOLS`.
- Deleted the two old-order tests (Letters→Numbers→Wingdings; "most-frequent color gets A").
- New invariants: `CURATED_SYMBOLS.length > 118` and all-unique; `CURATED_SYMBOLS[0]` is non-alphanumeric; no digits anywhere; glyph tier is letter-free and `> 118`; `SAFE_LETTER_SYMBOLS` equals the exact 19-letter list, contains none of B/G/I/O/Q/S/Z, and sits strictly after all glyphs (`slice(0, GLYPH_SYMBOLS.length)` === glyphs, `slice(GLYPH_SYMBOLS.length)` === letters).
- Frequency-desc + alphabetical tie-break tests kept (they reference `CURATED_SYMBOLS[0..3]`, auto-tracking the new lead glyphs).
- B4 overflow test rewritten to derive from `poolSize = CURATED_SYMBOLS.length` with `codeCount = poolSize + 20`; asserts full uniqueness, first `poolSize` colors keep single glyphs, and index `poolSize` triggers `` `${CURATED_SYMBOLS[0]}1` ``.
- `symbolFontPx` / `getContrastColor` tests unchanged.

## Glyph-Safety Approach

Per the glyph-safety constraint ("correctness of rendering beats pool size"), the selection was conservative: the shipping production base was preserved verbatim as the proven-safe core, the expansion drew only from well-supported text-default BMP blocks (Geometric Shapes U+25xx, math-operator U+22xx, Arrows U+21xx, text-default Dingbats U+27xx), all explicitly emoji-prone glyphs from the plan's exclude list were omitted, no astral/SMP code points were used, and near-identical look-alikes were deduped (small filled triangles vs the large ones; redundant diamond/lozenge outlines; dotted-outline circle vs plain circle). The plan's required in-browser visual check (load a 100+ color image, confirm distinct glyphs render with no tofu/emoji and legend matches grid) is the one item that cannot be exercised headlessly here and is carried to the orchestrator's post-execution in-browser verify, per verification note (e).

## Deviations from Plan

None affecting behavior. Within the planner's explicit "append, then dedupe near-identical pairs; drop any that render as tofu/emoji" guidance, four small triangles (`▴ ▾ ◂ ▸`) and several redundant diamond/lozenge/dotted-circle outlines were dropped as look-alikes during the initial build (glyph tier 122). The coordinator's post-execution in-browser distinctness audit then pruned a further 17 confusables (rotations + tiny-interior twins), taking the glyph tier to 105 and deliberately below the 118 alone-cover floor — the rarest colors fall to the unambiguous letter tail, which is the approved "letters as last resort." `CURATED_SYMBOLS` remains 124 (> 118). This is the intended curation latitude, not a spec change.
