## Task 1 — Fixed viewport (removes the top gap) + pinned hint

**Files:** `src/App.tsx` (Zone-2 refine content), `src/features/wizard/CanvasWorkspace.tsx`

Root cause: the Zone-2 content wrappers use `min-h-full`, so the tall right rail
(RefineScreen) stretches the whole row and Zone 2 scrolls; the canvas `main`
grows to match and centers the canvas with large empty vertical gaps (the "gap"
above the image). The pan/zoom hint (`absolute bottom-4`) then sits at the bottom
of that over-tall region instead of pinned to the visible viewport.

Fix: bound the refine row to Zone-2 height (`h-full` + `min-h-0`, not
`min-h-full`) and let the RefineScreen rail scroll internally, so the canvas
`main` stays a fixed-height viewport that fills the available height. The
`overflow-hidden` viewport then clips pan/zoom and the `absolute bottom-4` hint
pins to the true viewport bottom (item 3), visible while zooming.

Guard: do NOT regress the `@max-[640px]` mobile path (sticky 45dvh canvas +
internally-scrolling column). Keep `print:*` classes intact.

- **verify:** live preview — portrait image on Refine shows no top gap, canvas
  fills the viewport height, hint pill pinned to viewport bottom and stays put
  while zooming in. `npx tsc --noEmit`.
- **done:** screenshot confirms no gap + pinned hint; tsc clean.

## Task 4 — Visible slider track

**File:** `src/ui/Slider.tsx`

`appearance-none` currently leaves the range track invisible (only the thumb
shows). Add an explicit visible track (light "white" bar) + a defined accent
thumb via `::-webkit-slider-runnable-track` / `::-moz-range-track` and the thumb
pseudo-elements, so the color-count control reads as a slider.

- **verify:** live preview — the color-count control shows a white track bar with
  an accent thumb; drag still updates the color count.
- **done:** screenshot; `npx tsc --noEmit`.
