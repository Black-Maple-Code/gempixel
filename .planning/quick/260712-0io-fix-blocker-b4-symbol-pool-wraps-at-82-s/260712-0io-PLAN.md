# Quick Task 260712-0io — Fix Blocker B4 (duplicate legend symbols)

## Problem

`src/engine/symbols.ts` `generateSymbolAllocation` assigned
`CURATED_SYMBOLS[index % CURATED_SYMBOLS.length]` to each frequency-ranked color.
`CURATED_SYMBOLS` has exactly 82 glyphs (26 letters + 10 digits + 46 wingdings), so
with more than 82 active colors — the normal case for the 200-color kit and `'all'` —
the modulo wrapped: the 83rd-ranked color reused `'A'`, colliding with the most-frequent
color. The exported chart/legend (`export.ts`) and the viewer symbol overlay (`viewer.ts`)
then showed the **same glyph for two different DMC codes**, making the chart ambiguous and
un-stitchable — a defect in the tool's core deliverable (REVIEW-engine.md CR-01 / B4).

## Fix

Guarantee a unique symbol for every distinct color at any count:

- index `< 82` → single curated glyph (unchanged for ≤82 colors; preserves existing tests).
- index `>= 82` → deterministic multi-char symbol `base + suffix` where
  `base = CURATED_SYMBOLS[index % 82]` and `suffix = Math.floor(index / 82)` (≥ 1).
  A suffix ≥ 1 guarantees no clash with the single-glyph (suffix-0) set, and distinct
  indices produce distinct `(base, suffix)` pairs → all-unique symbols for arbitrarily
  many colors.

Rendering legibility: added `symbolFontPx(basePx, symbol)` in `symbols.ts` — returns
`basePx` for single glyphs and `round(basePx / symbol.length)` (min 1) for multi-char
symbols so a 2-char overflow symbol still fits the box a single glyph occupies. Applied at
all four symbol draw sites: `viewer.ts` overlay (font now set per-cell) and the three
`export.ts` sites (grid, combined sheet, legend swatch). Correctness (uniqueness) is the
priority; the font down-scale is the minimal cosmetic accommodation.

## Tasks

1. **Engine + render** — `symbols.ts` (allocation + `symbolFontPx`), `viewer.ts`,
   `export.ts`. Overflow colors get unique multi-char symbols; draw sites shrink the font
   for multi-char symbols.
2. **Tests** — `symbols.test.ts`: replace the old wraparound assertion with a
   >82-colors-all-unique test (200 colors → 200 unique symbols; first 82 keep single
   glyphs; index-82 = `'A1'` ≠ `'A'`), and a `symbolFontPx` scaling test.
