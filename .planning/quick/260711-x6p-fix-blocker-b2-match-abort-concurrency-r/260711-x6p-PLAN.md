<objective>
Fix blocker B2 (REVIEW-worker-data.md CR-01): the match-abort concurrency race. The worker's
abort flag is a single module boolean that the next match resets to false, so a superseded
in-flight run is never aborted, runs to completion, and — because worker-client reassigns
`onmessage` per call — delivers its stale, wrong-dimension `matches` array to the newest
`onComplete`. That array is then fed into `smoothMatches(matches, cols, rows)` at the current
dimensions, corrupting the grid and supply counts.

Replace the reset-a-shared-flag mechanism with a monotonic per-run id: the client stamps each
match with a runId, the worker adopts it as `currentRunId` (superseding older runs) and echoes
it on every reply, and both sides drop anything that does not match the live runId.

Purpose: Overlapping / rapid re-matches (grid-size slider drag, fast palette toggling) must
never apply a stale result to the current grid.
Output: runId-guarded worker + client, plus a regression test proving the superseded result is
ignored.
</objective>
