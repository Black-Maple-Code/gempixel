# Phase 260712-05k Plan 01: Fix Blocker B3 + Warning W9 (projectStore) Summary

Replaced `projectStore.save()`'s silent neighbor-eviction quota loop with a discriminated `SaveResult` status and switched `generateUUID()` from `Math.random()` to a CSPRNG, then wired a single text-only storage-full warning banner into `App.tsx` and gated the wizard's success message so a quota failure no longer destroys saved commissions or shows a false "Saved".

## What Was Built

**Task 1 — projectStore hardening (B3/CR-02, W9/WR-02):** commit `65f3b1a`
- Added exported `type SaveResult = { ok: true } | { ok: false; reason: 'quota' }`.
- `save()` now upserts the registry in place, writes the blob + registry inside a single `try`, and returns `{ ok: true }`; on `QuotaExceededError` it `console.error`s and returns `{ ok: false, reason: 'quota' }` without throwing and **without removing/overwriting any other project**. The `while (true)` eviction loop and victim `removeItem` are gone.
- `generateUUID()` now returns `crypto.randomUUID()` when available, else builds an RFC-4122 v4 id from `crypto.getRandomValues(new Uint8Array(16))` (version nibble → 4, variant bits → 10xx). No `Math.random` anywhere.
- Tests: replaced the eviction test with a no-data-loss quota test (asserts `{ ok:false, reason:'quota' }` and that a pre-existing `'old'` project still lists and `load('old')` deep-equals its data); added a 5000-id uniqueness + v4-shape test; added an explicit `{ ok:true }` success assertion. `recents.save()` and all other functions untouched.

**Task 2 — surface the warning (B3/CR-02):** commit `e75a7f5`
- `App.tsx`: added `saveErrorMsg` state; `handleSaveProject` now returns `boolean`, clears the prior warning at the top, captures `projectStore.save()`'s result, and on `!result.ok` sets the warning and returns `false` **before** the `setProjectsRegistry` / `setActiveProjectId` / `setSaveModalOpen(false)` side effects (returns `true` on success). Added one fixed, top-center, `no-print` rose banner at `z-[60]` (above the z-50 Save modal), text-only.
- `Step4Export.tsx`: prop type `handleSaveProject` → `(name, forceNewId?) => boolean`; all three save buttons now call `if (handleSaveProject(...)) showSaveSuccess();` so no false success on quota failure. The Save Project Modal's button ignores the boolean and correctly stays open on failure (no `setSaveModalOpen(false)` on the quota path).
