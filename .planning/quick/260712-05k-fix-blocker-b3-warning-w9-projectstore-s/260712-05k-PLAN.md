Fix blocker B3 (CR-02) and warning W9 (WR-02), both in `src/engine/projectStore.ts`, plus the minimal `App.tsx` / `Step4Export.tsx` wiring needed to surface a storage-full warning instead of silently destroying data.

- **B3 / CR-02:** `projectStore.save()` currently catches `QuotaExceededError` by evicting the oldest OTHER project (removeItem + drop from registry) in a retry loop — permanent, unannounced data loss. Replace with a discriminated return status; never evict another project.
- **W9 / WR-02:** `generateUUID()` uses `Math.random()`; the id is the localStorage key, so a collision overwrites a project. Switch to a CSPRNG (`crypto.randomUUID` with a `crypto.getRandomValues` v4 fallback).
- **Wiring:** the single `save()` call site in `App.tsx` must react to the new status and show a user-visible warning; the wizard save UI must not show a false "Saved" message on failure.

Purpose: Eliminate two silent data-loss paths for saved commissions while keeping storage keys and serialized shapes frozen (existing projects must keep loading).
Output: Updated `projectStore.ts` + tests; a targeted quota-warning banner in `App.tsx`; a guarded success message in `Step4Export.tsx`.
</objective>
