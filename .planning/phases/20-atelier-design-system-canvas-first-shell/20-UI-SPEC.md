Registry safety is not applicable.

**Font wiring (DESIGN-02, D-09/D-10/D-11):**

| Face | Package | `@theme` override | Role |
|------|---------|-------------------|------|

- No-CLS: `font-display: swap` + Fontaine Vite plugin auto-generating fallback metric overrides (`size-adjust`/`ascent-override`/`descent-override`). No hand-authored metrics, no `font-display: optional`.

## Spacing Scale

8px-based rhythm (design handoff §Spacing). Declared token values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Small chip radius gaps, mono micro-label inline padding |
| sm | 8px | Control gaps, step-nav item internal spacing |
| md | 16px | Default element spacing, top-bar padding |
| lg | 24px | Card padding, section padding |
| xl | 32px | Major inter-section gaps |
| 2xl | 48px | Layout breaks |
| 3xl | 64px | Page-level spacing |

**Exceptions (layout dimensions, not spacing tokens — carried verbatim from the handoff):**
- Desktop card frame: fixed **1180px** wide.
- Refine rail: 360px · Supplies order-summary: 320px · Order left column: 470px (referenced by Phase 23; shell reserves the structure).
- Card padding renders at the handoff's 22–28px band; **normalize to 24px (`lg`)** unless a frame in the prototype demands otherwise.
- Control gaps 8–14px → snap to 8px (`sm`); mobile phone frame 300×620 is Phase 24.

## Typography

Two system weights: **400 (regular)** and **600 (semibold)**. Mono data codes use **700** as a documented data-emphasis exception (DMC codes, bag counts).

| Role | Face | Size | Weight | Line Height | Notes |
|------|------|------|--------|-------------|-------|
| Display | Newsreader | 34px | 600 | 1.2 | Screen titles (e.g. "Photo → Diamond chart") |
| Heading | Newsreader | 18–21px | 600 | 1.2 | Section titles ("Order summary" 18), wordmark 21 |
| Body | Archivo | 16px | 400 | 1.5 | Default UI + prose; 14px allowed for dense secondary UI |
| Label | JetBrains Mono | 10–11px | 400 | 1.4 | UPPERCASE micro-labels, letter-spacing 0.08–0.14em, color Ink faint `#9A927D` |

Data emphasis exception: JetBrains Mono **700** for DMC codes and green bag counts ("N×200").

## Color

60/30/10 split on the Atelier light palette (the only theme — dark mode fully retired, DESIGN-01).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F4F1E9` | Page + card background (`--bg`); with surfaces `#FCFAF4` raised, `#FFFFFF` inputs/chart mounts |
| Secondary (30%) | `#FCFAF4` | Top bar, refine rail, order-summary panel, docks (`--panel`); borders `#E4DECF` / `#EEE8D9` |
| Accent (10%) | `#0E6E5C` | See reserved-for list below (`--accent`) |
| Destructive | `#C2410C` | Needs-attention / reorder only — **no destructive action exists in Phase 20** (chrome only); reserved for later phases |

**Ink scale:** primary text `#1B1A17`, muted `#6B6459`, faint `#9A927D`.
**Warn (amber):** `#A85A11` on `#F7EFD8` — in-progress/caution (not used by Phase 20 chrome).

**Accent (`#0E6E5C`) reserved for — explicit list, never "all interactive elements":**
1. Primary CTA buttons and the pill/button green fills.
2. The **current** step nav circle (filled green + number + bold ink label) and **completed** step circle (filled green + check).
3. Step-nav connectors that have been **passed** (green line; `#D8D0BC` when ahead).
4. Selected state — 2px green border + `#EAF2EF` (accent tint) fill on selected cards.
5. Key figures — est. total, drill counts, green bag counts in mono.
6. Focus ring on interactive chrome.

Top-bar "Save" pill is the one deliberate dark accent: `#1B1A17` bg, `#F4F1E9` text (not the green accent).

**Chart cell colors are data-driven (DMC palette), NOT theme tokens** — they come from the match and are exempt from the 60/30/10 accounting.

## Interaction Contract (canvas-first shell)

**Navigation (SHELL-01, D-03):** one horizontal 4-step bar `1 Upload · 2 Refine · 3 Supplies · 4 Order` is the **only** navigator. Both legacy nav surfaces — the desktop footer dot-nav and the mobile fixed bottom bar — are deleted. No sidebar, hamburger, or page-flip wizard. Forward via the step's primary CTA; back by tapping a completed step. Labels come from a single `STEP_META` map (D-02).

**Step-nav visual states (handoff chrome spec):**

| State | Circle | Label | Connector |
|-------|--------|-------|-----------|
| Completed | Green filled + check | Muted `#6B6459` | Green (passed) |
| Current | Green filled + number | Bold ink `#1B1A17`, `aria-current="step"` | — |
| Upcoming (locked) | Outlined `#9A927D` | Muted, container `opacity:.45` | `#D8D0BC` (ahead) |

**Gating (SHELL-02, D-12):** locked/upcoming steps render `aria-disabled="true"`, are removed from tab order, and show a short static tooltip. All four stay visible as a stable journey map; a locked tap never dead-ends (no toast, no navigation).

**Soft-invalidate + recompute (D-13):** editing a completed upstream step marks downstream steps `stale`, **keeps last-good match/supplies on screen**, blocks advancing past a stale step, and surfaces one "Recompute match" banner CTA. No silent worker re-fire; the expensive match runs once on intentional click. Follows the existing `actionError`/banner surfacing convention.

**Single-mount viewer (SC4, D-14):** the one `<CanvasViewer>` is hoisted to shell scope and is **never** inside a `{step===n && …}` branch. The four step panels are CSS-toggled siblings (`hidden`/`display:none`). Zoom/pan/LOD + in-flight worker state survive step changes.

## Copywriting Contract

Phase 20 introduces only chrome copy; screen-body copy ("Match colors →", "Continue to order →", "Place order") stays with the legacy Step bodies until Phase 23.

| Element | Copy |
|---------|------|
| Step nav labels | `Upload` · `Refine` · `Supplies` · `Order` |
| Wordmark | `GemPixel` (Newsreader 21/600, beside the green 3×3 pixel-tile mark) |
| Top-bar primary chrome action | `Save` (dark pill) |
| Locked step tooltip — Refine | `Upload an image to unlock` |
| Locked step tooltip — Supplies | `Compute a match to unlock` |
| Stale/recompute banner heading | `This step is out of date` |
| Stale/recompute banner CTA | `Recompute match` |
| Empty state | Owned by the legacy Upload (Step1) body, unchanged this phase — not re-specified here |
| Error state | Existing defensive `actionError` banner convention (non-throwing; match/canvas/storage failures degrade, never crash) |
| Destructive confirmation | None — Phase 20 has no destructive action |
