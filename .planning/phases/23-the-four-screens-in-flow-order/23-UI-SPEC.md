# Phase 23 — UI Design Contract

> Visual and interaction contract for the four customer screens (Upload → Refine → Supplies → Order). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **This phase COMPOSES an already-implemented, LOCKED design system.** The Atelier tokens (`src/index.css` `@theme inline`) and the `src/ui/` primitive APIs (SegmentedControl, Slider, SizeCard, Pill, Button, Card chrome) were built in Phases 20–21 and are frozen. This contract governs per-screen **layout, interaction states, copywriting, and empty/loading/error states** — it does NOT re-derive the palette, typography, or primitive APIs. ## Spacing Scale

Atelier is an **8px-based** rhythm. Declared canonical scale (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Chip inner padding, icon gaps, symbol-cell gaps |
| sm | 8px | Compact control gaps, pill padding, table cell vertical padding |
| md | 12px | Control-group gaps inside rails |
| lg | 16px | Default element spacing, list-row gaps |
| xl | 24px | **Card padding (canonical)**, section padding |
| 2xl | 32px | Major section breaks within a screen |
| 3xl | 48px | Screen top/bottom breathing room |

**Exceptions (locked by the Atelier design contract / handoff — do NOT "correct" to a multiple of 4):**
- Card padding renders in the handoff's **22–28px** band; **use 24px** as the single canonical card padding.
- Control gaps span **8–14px** in the handoff; snap to **8px or 12px** (avoid 14px in new work).
- Radius tokens are pre-tokened and locked: `--radius-card: 12px`, `--radius-control: 8px`, `--radius-pill: 20px`, small chips 4–6px. (Handoff notes controls "7–8px" → use the implemented **8px**.)
- Fixed layout dimensions (from Storyboard A, not on the 4px grid — keep verbatim): frame **1180px**; Refine rail **360px**; Supplies order-summary panel **320px**; Order left proof column **470px**; Upload dropzone **560×250** (radius 14px, `2px dashed #C9BFA6`).
- Card shadow (locked token): `--shadow-card: 0 40px 80px -30px rgba(0,0,0,0.5)`.

## Typography

Locked Atelier type system (three families). Roles below — **screens must use these roles, not ad-hoc sizes.** Two primary weights (400 regular + 600 semibold); display headings may reach 700 as a locked Atelier exception.

| Role | Family | Size | Weight | Line Height | Applied to |
|------|--------|------|--------|-------------|-----------|
| Display | Newsreader (serif) | 34px | 600 | 1.15 | Upload title "Photo → Diamond chart" |
| Heading | Newsreader (serif) | 18–21px | 600 | 1.2 | Panel titles ("How big should it be?", "Drill supply plan", "Order summary"), top-bar wordmark (21px) |
| Body | Archivo (sans) | 14–16px | 400 / 500 | 1.5 | Subtitles, captions, control labels, address text |
| Micro-label | JetBrains Mono | 10–11px | 500 | 1.4 | UPPERCASE section labels, letter-spacing 0.10em, color `--faint` `#9A927D` ("EDGE CLEANUP", "COLOR COUNT · 24 of 26 matched", "DMC COLOR", "SENT TO THE LAB · AUTO-FILLED 4/4") |
| Data / number | JetBrains Mono | 12–14px | 600 (emphasis) | 1.3 | Drill counts, bag counts ("9×200"), grid dims ("80×53 grid"), prices, DMC codes (mono bold) |

Notes:
- Key **figures** (drill counts on size cards, "Est. total", bag counts, prices) render in accent green `#0E6E5C` (see Color).
- DMC code = mono bold ink; color **name** beside it = Body 14px muted (`--muted` `#6B6459`).

## Color

Atelier light — the only theme. 60/30/10 split (values already live in `src/index.css`):

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F4F1E9` (`--bg`) | App/page background AND card background |
| Secondary (30%) | `#FCFAF4` (`--panel`) + `#FFFFFF` (`--panel-2`) | Raised surfaces: top bar, Refine rail, Supplies summary panel, Order left proof column (`--panel`); inputs, chart mounts, inner cards (`--panel-2`) |
| Accent (10%) | `#0E6E5C` (`--accent`) | See reserved-for list below |
| Accent tint | `#EAF2EF` | Selected-card fill, "ok" pill background (paired with accent border/text) |
| On-accent text | `#F4F1E9` (`--on-accent`) | Text/icon on green surfaces |
| Warn (amber) | `#A85A11` (`--warn`) | Caution / stale / "out of date" affordances (dark amber reads on cream) |
| Alert (orange) | `#C2410C` | Needs-attention (primarily ops console; rare in customer flow) |
| Text ink | `#1B1A17` (`--ink`) | Headings, body |
| Text muted | `#6B6459` (`--muted`) | Secondary text, color names |
| Text faint | `#9A927D` (`--faint`) | Micro-labels, meta, mono captions |
| Border | `#E4DECF` (`--border`) | Card + control borders |
| Border subtle | `#EEE8D9` (`--border-2`) | Table row dividers |

**Accent (`#0E6E5C`) reserved for — explicit list (never "all interactive elements"):**
1. Primary CTA buttons ("Browse files", "Match colors →", "Continue to order →", "Download order packet").
2. Active/current step in the StepBar (filled green circle; completed = green check).
3. Selected state: size-card 2px green border + `#EAF2EF` fill + "BEST" tag; selected segment in SegmentedControl; selected finish card on Order.
4. Slider active track + handle.
5. Key figures: size-card drill counts, bag counts ("9×200"), "Est. total" / "Total", per-line prices.
6. Inline links styled as accent ("Custom size").

**Destructive / semantic:**
- **No dark red destructive palette** is introduced. The one destructive action (remove a recent project) uses a **ghost/muted** control with inline confirmation, not a red button (see Copywriting).
- Warn amber is for **stale/estimate/caution**, not deletion.
- **Honesty rule (Phase 22 lineage):** an unpriced canvas (`canvasPriced:false`) or `hasUnpricedSize` color is **never rendered as `$0`** — surface it as "est./unavailable" via the existing banner + an "est." affordance on the line item.

## Per-Screen Layout & Interaction Contract

> These four sections are the executor's visual source of truth. Every value traces to the handoff README (Storyboard A) + the reference render. Screens are **pure/props-only** (Phase 23 D-01); App owns all state.

### A1 · Upload (`UploadScreen.tsx`)
**Layout:** centered single column inside the 1180px card.
- Display title "Photo → Diamond chart" + muted subtitle, centered.
- **Dropzone 560×250**, `2px dashed #C9BFA6`, bg `--panel` `#FCFAF4`, radius 14px, containing: a 3×3 pixel motif (gem colors), "Drag & drop, or" (Body muted), and the green **Browse files** button (`Button variant="primary"`).
- Below: a **RECENT** mono micro-label + a horizontal row of **project chips** — thumbnail mini-grid + project name (from `projectStore.list()` → `ProjectSummary[]`, has thumbnails). Selecting a chip calls `loadProject(id)` → rehydrates state (no worker run) → advances into the flow.
- **Canvas-size selection is NOT here** (moved to Refine, SC1/D-10).

**Interaction states (dropzone):**
| State | Visual |
|-------|--------|
| Idle | `2px dashed #C9BFA6`, bg `--panel` |
| Hover / drag-over | Border → accent `#0E6E5C`, bg → `#EAF2EF` tint; cursor copy. (Drives `isDragOver`.) |
| Error (bad type) | Border → warn `#A85A11`; inline error copy below the zone (see Copywriting). |
| Recent chip hover | Border `--border` → accent; subtle lift. |
| Recent chip remove | Ghost "Remove" → inline "Remove? Yes / Cancel". |

### A2 · Refine (`RefineScreen.tsx`) — **KEYSTONE**
**Layout:** two columns inside the card. Left = live chart preview (the single-mount `<CanvasViewer>` stays in `<main>`; screen does NOT own it). Right = a **360px refine rail** (`border-left`, bg `--panel`), always open (never a drawer). Rail top→bottom:
1. **"How big should it be?"** — vertical stack of **SizeCards** (`src/ui/SizeCard`), each: label + inch dims (Heading-ish) · mono grid size (`{cols}×{rows} grid`) · **drill count** right-aligned in accent green. Presets read as recommendations; **selected** = 2px green border + `#EAF2EF` fill + "BEST" tag (`Pill variant="tag"`). Footer: "Bigger canvas = more detail & more drills. **Custom size**" (accent link toggles a custom cols/inches entry with sane clamps, reusing the existing App width/height clamp precedent).
   - Inches/drill count are **computed in App** via `gridToInches`+`formatInches` and passed as props (no derivation in the card — 23-RESEARCH Pattern 2). Never a hard-coded mock label.
2. **Edge cleanup** — `SegmentedControl` 4-seg `Off / Light / Med / Strong` → smoothing strength 0–3. Micro-label "EDGE CLEANUP · smooths ragged edges".
3. **Color count** — `Slider` (native range, `onInput` live). Micro-label header "COLOR COUNT · {current} of {detected} matched". **`max = detectedColorCount`** (floor 8); handle near the top by default. Caption per Copywriting. Live merged-count readout ("{current}") sits beside the slider so the smoothing dead-zone reads as "already at N", not a broken control.
4. **Advanced** disclosure (native `<details>`, closed by default): kit (100/200/all, default **all**), color-exclusion, drill-shape (square/round, default **square**). Uses the existing `.caret-icon` rotate animation.
- Rail footer: green **Match colors →** button.

**Two-tier reactivity (D-03/D-04) — the keystone behavior contract:**
| Control | Tier | Feedback |
|---------|------|----------|
| Size selection (card / custom) | **Worker re-match** | Card drill counts + inches update **instantly** (pure). The full-grid preview updates **on Recompute** — a size change marks downstream steps stale (StepBar amber marker) + surfaces the single "Recompute match" CTA (Phase 20 D-13). Must NOT re-fire the worker per click (avoids B2 abort-race). |
| Edge cleanup + color slider | **Main-thread post-process** | **Live, every tick** — re-renders the chart with no worker re-fire, no staleness. |

**Loading state (during Recompute):** canvas shows the last-good grid; the "Recompute match" CTA shows a spinner; StepBar downstream markers read stale (amber) until the worker returns and `detectedColorCount` refreshes.

**Slider interaction states:** idle handle `--accent`; focus = visible ring; drag = live `onInput` (never `onChange` — Preact freezes mid-drag); disabled (no image) = muted, non-interactive. Max must NOT jump under the user (keyed to `detectedColorCount`, raw-match-stable).

### A3 · Supplies (`SuppliesScreen.tsx`)
**Layout:** two columns. Left = **drill supply plan table**; right = **320px order-summary panel** (`border-left`, bg `--panel`).
- **Supply table** — header mono micro-labels `(symbol) · (swatch) · DMC COLOR · DRILLS · BAGS`. Each row: symbol chip (from `symbolMap[code]`) · color swatch (hex from `DMC_PALETTE`) · DMC code (mono bold) + name (Body muted) · drills incl. **+10% safety** · bags (accent mono, e.g. "9×200"). Caption "counts include +10% safety". Row divider = `--border-2`.
- **"Why these bags?"** — inline disclosure (native `<details>`), body = `DYE_LOT_WHY_SENTENCE`.
- **Order-summary panel** — Heading "Order summary", then mono line items rendered **verbatim** from `buildOrderQuote(...).lineItems` (Total drills, Bags (200-ct), Drill packets, Canvas print · {size}, Shipping (est.), Tax) → divider → **Est. total** in accent green 20px = `.totalCents`. Footer note "Billed by GemPixel · printed & shipped by our lab". **Zero local total math** (D-07) — `.reduce` on cents in a screen is forbidden.
- **Honesty:** unpriced canvas / unpriced size → "est."/"unavailable" affordance, never `$0`.

### A4 · Order (`OrderScreen.tsx`)
**Layout:** full-width app-shell card, two columns.
- **Left (470px, bg `--panel`):** "YOUR CANVAS PROOF" — chart proof with a green size badge (e.g. "24 × 16 in"). Then "SENT TO THE LAB · AUTO-FILLED 4/4" — a 4-row **locked spec** list: IMAGE (Finished chart · print-ready) · PRODUCT (Rolled Canvas — **LOCKED** pill, `Pill variant="neutral"`) · SIZE ({w}×{h} in · from {cols}×{rows} grid, via `gridToInches`) · FINISH ({selected}). Caption "Everything the print lab needs is carried over. Nothing to re-enter." Spec is **auto-filled and non-editable** (ORDER-01) — nothing GemPixel already resolved is re-entered.
- **Right (bg `--panel-2`):** **CANVAS FINISH** — two selectable cards (**Trimmed** default + "BEST FOR ART" tag; **Image wrap** alt). **SHIP TO** — name + address card + EDIT (client-side only, embedded in packet, never sent). **Price** card — same `buildOrderQuote` line items + green **Total**.
- **Primary action:** green **Download order packet** → serializes a versioned, self-contained JSON packet (schema `version` field; design/chart ref, locked canvas spec, optimized gem-bag list from `OrderSupplyPlan.rows`, integer-cents `OrderQuote` snapshot, client-only ship-to). `packetId` via `generateUUID()` (CSPRNG). Reuse the `export.ts` anchor+Blob download idiom. Filename `gempixel-order-<packetId-short>.json`.
- **Terminal state (honest, D-09):** "Packet downloaded — take this to the vendor." **No order number, no receipt, no payment UI.** The mock's "Place order · $57.00" is explicitly NOT shipped.

**Finish-card interaction states:** idle = `--border`; hover = border accent; **selected** = 2px accent border + `#EAF2EF` fill + "BEST FOR ART" tag on Trimmed; keyboard-selectable.
