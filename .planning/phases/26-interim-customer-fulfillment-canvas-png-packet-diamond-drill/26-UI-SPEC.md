# Phase 26 — UI Design Contract

> Visual and interaction contract for the Order-step interim fulfillment work
> (ORDER-04 canvas PNGs + ORDER-05 Diamond Drills USA cart + the D-08 error-banner
> re-token). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> This phase adds **zero net-new visual language**. It composes the existing
> Atelier light design system (`src/index.css`) and the hand-built `Button` / `Pill`
> primitives into two honest task sections on the already-shipped `OrderScreen`.
> The contract below is therefore a *conformance* contract: the new UI must reuse
> the exact tokens, primitives, and copy patterns the Order step already ships.

## Spacing Scale

The project uses the Tailwind v4 default 4px base (documented as the "8px spacing"
system in DESIGN-02). Only these steps appear on the Order step and must be reused —
every value is a multiple of 4.

| Token | Value | Order-step usage |
|-------|-------|------------------|
| gap-1 / py-0.5 area | 4px | Label→value stacks, pill inner gap |
| gap-2 / p-2 | 8px | Compact control spacing, ship-to field grid gap, download-button column gap |
| gap-3 / p-3 | 12px | Spec-row vertical padding, finish-card padding |
| gap-4 / p-4 | 16px | Terminal/done-state panel padding, intra-section stacks |
| gap-6 / p-6 | 24px | **Section-level padding and the gap between the two task sections** |

Radius tokens (reuse, do not redefine): `--radius-card: 12px` (section cards, finish
cards, done-state panels), `--radius-control: 8px` (inputs, buttons), `--radius-pill: 20px`
(pills, size badge).

**Exceptions:** none. Do not introduce any spacing value that is not already in the
table above. The two new task-section headers reuse the same `p-6` section padding the
existing left/right OrderScreen columns already use.

## Typography

Exactly four roles are in play on the Order step, two weights. Reuse the existing
utility classes — do not add font sizes.

| Role | Size | Weight | Line Height | Utility / usage |
|------|------|--------|-------------|-----------------|
| Eyebrow label | 10px | 600 (semibold) | 1.5 | `font-mono text-[10px] uppercase tracking-wider text-faint` — section eyebrows ("Get your canvas made" kicker, "Order your drills" kicker, field-group legends) |
| Body / helper | 12px | 400 (regular) | ~1.625 (`leading-relaxed`) | `text-xs text-muted` — sub-terminal explainer copy, ship-to hint |
| Body / value | 14px | 400 reg / 600 semibold | 1.5 | `text-sm text-ink` — spec values, finish labels (semibold), button labels |
| Emphasis figure | 20px | 600 (semibold) | 1.2 | `font-mono text-[20px] font-semibold text-accent` — the quote total (already present; keep) |

Weights allowed this phase: **400 (regular)** and **600 (semibold)** only. Section
*headings* for the two task groups use the eyebrow-label recipe (10px mono uppercase),
matching every other titled group on the screen — do **not** introduce a serif display
heading inside the Order card (Newsreader is shell-chrome/wordmark only here).

## Color

Atelier light, single skin. 60/30/10 split as it already renders on the Order step.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--bg` #F4F1E9 (cream) | App viewport background behind the Order card |
| Secondary (30%) | `--panel` #FCFAF4 + `--panel-2` #FFFFFF; borders `--border` #E4DECF / `--border-2` #EEE8D9 | Section ① canvas card on `bg-panel`, section ② drills card / right column on `bg-panel-2`, dividers |
| Accent (10%) | `--accent` #0E6E5C (deep green) | See reserved-for list below |
| Accent tint | #EAF2EF (documented green handoff tint; arbitrary value, not a new token) | Selected finish card, done-state confirmation panels, size badge |
| Caution / error | `--warn` #A85A11 (dark amber) | The re-tokened `actionError` banner (D-08) — the ONLY semantic-2 color this phase |

**Accent (`--accent` #0E6E5C) reserved for — explicit list:**
- Primary download CTAs (canvas grid PNG, grid+legend PNG, legend PNG, JSON packet) — `Button variant="primary"`
- The single Diamond Drills USA cart CTA (section ②) — `Button variant="primary"`
- The quote **total** figure
- Selected finish-card border (`border-2 border-accent`) + its #EAF2EF tint
- Input focus border (`focus:border-accent`)
- Per-task **done-state** panels (border-accent on #EAF2EF) — the "Downloaded ✓" and "Cart opened ↗" sub-terminals
- The success glyph/text in each sub-terminal

Accent is **not** used for: body text, section chrome, borders on idle cards, or the
error banner. No true-red destructive color exists in this palette and none is added —
see Copywriting (no destructive actions this phase).

**D-08 grep-gate (hard rule):** after the re-token, no `bg-slate-9(00|50)`, no
`text-white`, and no `rose-950` may remain on any live (non-print) surface. The
`actionError` banner moves to the Order step and re-tokens to a light Atelier
treatment: `border border-warn text-warn bg-panel-2` (dark-amber caution on a light
panel), NOT the retired dark-slate/rose banner. Print-mode `@media print` rules in
`index.css` are out of scope for the grep-gate (they are print-only, not live UI).

## Copywriting Contract

The Order step is **two errands to two vendors** (D-06/D-07). Copy must stay honest:
files really land on disk ("downloaded"); the external cart can only be *opened*, never
confirmed ("cart opened", never "ordered"). Discretionary wording below is fixed here so
the executor has no ambiguity; the planner may refine within the honest-handoff constraint.

| Element | Copy |
|---------|------|
| Section ① heading (eyebrow) | **Get your canvas made** |
| Section ① sub-copy | Download your artwork and spec, then send them to your canvas maker. |
| Section ① CTA — grid PNG | **Download canvas (grid)** |
| Section ① CTA — combined | **Download grid + legend** |
| Section ① CTA — legend | **Download legend** |
| Section ① CTA — JSON packet (existing) | **Download order packet** |
| Section ① done-state | **Downloaded ✓** — heading; sub: "Your canvas files are on your device — send them to your canvas maker." |
| Section ② heading (eyebrow) | **Order your drills** |
| Section ② CTA | **Open drill cart at Diamond Drills USA ↗** (opens external, new tab) |
| Section ② done-state | **Cart opened ↗** — heading; sub: "Finish your order on Diamond Drills USA." |
| Primary CTA (phase headline) | **Download canvas (grid)** / **Open drill cart at Diamond Drills USA ↗** — two honest primaries, one per section (no single "Place order") |
| Empty state | Not reachable — the Order step is validation-gated (no Supplies/Order without a computed match, SHELL-02). No empty-state copy required. |
| Unavailable line (existing) | Keep the existing `Pill variant="neutral"` "unavailable" for an unpriced canvas line; do not restyle. |
| Error state | Re-tokened `actionError` banner: "Couldn't prepare that download. Please try again." (or the App-provided message) — dark-amber `--warn` text on a light panel; never implies a failed *purchase*. |
| Destructive confirmation | **None.** This phase has no destructive action (no delete, no irreversible submit). No confirmation dialog is introduced. |

**Honesty guardrails (checker will enforce):**
- Never label the drills sub-terminal "Ordered", "Purchased", or "Complete" — only "Cart opened".
- Never imply a single GemPixel-placed order or any charge (D-06). Two vendors, two sub-terminals.
- The two done-states are **independent** — one section may be done while the other is not (per-task state replacing the single `packetDownloaded` boolean, D-07).

## Component Inventory (reuse-only)

No new primitive is created. The two task sections are composed from existing pieces:

| Component | Source | Role this phase |
|-----------|--------|-----------------|
| `Button variant="primary"` | `src/ui/Button.tsx` | All PNG/packet downloads + the cart link (accent, `rounded-[var(--radius-control)]`) |
| `Pill variant="neutral" / "ok" / "tag"` | `src/ui/Pill.tsx` | LOCKED spec pill, "unavailable" line, per-task done badges |
| Done-state panel pattern | existing OrderScreen terminal (`:258–267`) | Cloned into two per-task sub-terminals (border-accent on #EAF2EF) |
| Finish cards / spec `<dl>` / ship-to grid | existing OrderScreen | Unchanged — section ① wraps the existing spec + adds the PNG buttons; layout stays the two-column `md:flex-row` shell |
| `actionError` banner | App-owned, re-homed to Order (D-08) | Re-tokened to warn-on-light |

**Layout contract:** keep the existing `section[data-screen="order"]` two-column shell
(`flex flex-col gap-6 md:flex-row`). The two honest task sections live in the right
column as stacked groups separated by `gap-6`, each led by a 10px mono eyebrow heading —
mirroring the existing "Canvas finish" / "Ship to" / "Price" group rhythm. Do **not**
introduce a flat button stack (the retired `Step3Canvas` "Order & Actions" pattern, D-07).

## Interaction States

| Surface | Idle | Active/hover | Done (terminal) | Disabled |
|---------|------|--------------|-----------------|----------|
| Download button (×4) | accent bg, 14px label | `hover:brightness-110`; `active` translate(2,2) via `.btn-chunk` recipe | section collapses to "Downloaded ✓" panel once all canvas files fire | n/a (always enabled when match exists) |
| Cart button | accent bg, `↗` glyph | brightness hover | "Cart opened ↗" sub-terminal after click | n/a |
| Finish card | `border-border bg-panel` | `hover:border-accent` | `border-2 border-accent bg-[#EAF2EF]` when selected | — |
| Ship-to input | `border-border bg-panel` | `focus:border-accent` outline-none | — | — |
| Error banner | hidden | — | warn-on-light, dismiss/retry via re-action | — |

**Sequential-download caveat (D-03):** the multiple programmatic PNG downloads may trip
a one-time Chromium "Download multiple files?" prompt and can be dropped by Safari/Firefox
if fired too fast. Space them with small awaits (handlers are already `async`). This is a
functional detail, not a visual one — no spinner/progress UI is added for the interim path.
