<objective>
Add the additive `drawLegendOnly` renderer to `engine/export.ts` so the Order step can
download the color legend as its own PNG (the "legend separately" artifact of SC1), using
the exact swatch/symbol/label draw the combined sheet already ships. This is the ONE
sanctioned engine addition of an otherwise UI-focused phase (D-05): it adds a new export and
does not touch any Phase-22-frozen signature.

Purpose: SC1 needs three canvas PNGs — grid-only, grid+legend, and legend-on-its-own. The
first two reuse `drawCanvasOnly` / `drawCombinedCanvasSheet` verbatim; the third needs a
renderer that draws the legend band alone. A UI-layer crop of the combined sheet is rejected
(brittle — it re-derives private layout constants and captures stray full-width folding-guide
fragments), so the legend loop is factored into an artifact-free additive export instead.

Output: `drawLegendOnly` export + its unit coverage. No UI wiring in this plan (App wires it
in 26-02).

D-05 escape hatch (record if taken): if factoring the shared loop would force a signature
change on `drawCombinedCanvasSheet`, do NOT ship the crop — fall back to shipping only two
PNGs (grid, grid+legend) plus the existing `printLegendSheetOnly` print path, and record the
SC1 third-PNG trim explicitly in the SUMMARY. The preferred outcome is the additive renderer.
</objective>
