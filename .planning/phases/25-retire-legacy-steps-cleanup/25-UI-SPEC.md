# Phase 25 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope note:** Phase 25 is a strangler-close **cleanup** phase (delete legacy `Step1/2/4`, theme
> remnants, dead preset state) that also lands a batch of user-facing **UX / viewport / wizard-chrome
> refinements** (CONTEXT D-02..D-09). It builds on the **already-shipped Atelier design system**
> (Phases 20–24). This contract does NOT invent a new system — it locks the existing tokens as the
> source of truth and pins the layout/interaction contracts for the specific refinements. Every
> Phase 20–24 token decision is treated as **locked**; this doc re-states them only so the checker,
> planner, and executor have one place to validate against.

## Design System

| Property | Value |
|----------|-------|
| Tool | **none** (established custom "Atelier" system; shadcn deliberately N/A) |
| Preset | not applicable |
| Component library | none — hand-rolled Preact primitives in `src/ui/` (`SizeCard`, `SegmentedControl`, `Slider`, `Button`, `Pill`, `cn`) |
| Icon library | none — emoji glyphs (`➕ ➖ ⛶ ⚠️`) + pure-CSS marks (`.gem-logo`) + inline SVG carets |
| Font | Display: **Newsreader Variable** (serif) · Body/labels: **Archivo Variable** (sans) · Numeric/mono: **JetBrains Mono** |

**shadcn gate resolution — confirmed deliberate absence (not a prompt).** No `components.json`; stack is
Preact (not React) + Tailwind v4 with a CSS-custom-property token layer (`src/index.css`). CLAUDE.md /
GEMINI.md lock a **browser-native, zero-new-deps, "do NOT use full React"** constraint. shadcn is
React/Radix-based and would violate that constraint, so it is not initialized and the registry safety
gate is not applicable. This is intended, not a gap.

**Token source of truth:** `src/index.css` `:root` + `@theme inline`. Semantic Tailwind utilities
(`bg-bg`, `bg-panel`, `text-ink`, `text-muted`, `border-border`, `bg-accent`, `text-on-accent`,
`text-warn`) are the canonical way to reach a token. A legacy `slate-*/indigo-*/sky-*/violet-*/emerald-*`
→ token **remap shim** still exists in `@theme inline`; **D-09 + D-01 sweep the last consumers of that
shim** — new/edited code in this phase MUST use the semantic names, never the remapped scale names.

## Spacing Scale

The project uses Tailwind's default 4px step scale. Declared values in active use (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `p-1`) | Icon gaps, HUD segment padding, inline list rows |
| sm | 8px (`gap-2`, `p-2`, `px-2`) | Compact element spacing, pill padding |
| md | 12px (`gap-3`, `py-3`, `px-3`) | Header padding, control gaps |
| lg | 16px (`gap-4`, `px-4`, `p-4`) | Default section / panel padding, viewport HUD offset |
| xl | 24px (`gap-6`, `p-6`) | Rail section breaks |
| 2xl | 32px (`gap-8`) | Layout gaps |

**Radius tokens (locked):** `--radius-card: 12px`, `--radius-control: 8px`, `--radius-pill: 20px`.

**Phase-25 spacing exceptions / additions:**
- **Touch targets:** zoom + HUD buttons stay at **`min-h-[44px] min-w-[44px]`** (D-09 confirms/keeps the
  Phase 24 touch-target bump — already present in `CanvasWorkspace.tsx`; must survive the re-token).
- **Refine rail width cap (D-06):** rail is width-capped to **`max-w-[320px]`** (down from the current
  ~360px, which left dead space on the right and ate the viewport). Canvas viewport is `flex-1`. At
  ≥1024px the layout must fit the browser width with **no horizontal scroll and no rail encroachment on
  the canvas**. Exact px is planner's call within a 300–340px band; 320px is the prescribed default.
- **Fixed-shell zone gaps (D-05):** the three shell zones (top bar / scroll content / bottom action bar)
  are flush; internal content padding stays on the 4px scale. Bottom action bar padding matches the
  header (`px-4 py-3`) for visual symmetry.

## Typography

Fonts are locked (self-hosted via Fontaine). Roles in active use:

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Display / wordmark | 21px (`text-[21px]`) | 600 (semibold) | 1.0 (`leading-none`) | Newsreader (`font-display`) |
| Body / control label | 14px (`text-sm`) | 400 / 500 | 1.5 | Archivo (`font-sans`) |
| Small label / pill / uppercase eyebrow | 12px (`text-xs`) | 700 (bold), `uppercase tracking-wide` | 1.4 | Archivo |
| Micro / hint / tooltip / legend | 10px (`text-[10px]`) | 500–700, `uppercase tracking-wider` on switch labels | 1.5 | Archivo / JetBrains Mono (counts) |

**Weights:** two-weight discipline for text — **regular (400)** for body, **semibold/bold (600–700)** for
emphasis, labels, and CTAs. No new weights introduced this phase.

**Phase-25 note:** the view-switcher segment labels ("Grid Colors" / "Grid + Symbols" / "Original Photo")
and the drills hint stay at the 10px uppercase micro role. The "recomputing…" pending label reuses the
existing loading-overlay type (`text-sm font-medium`, 14px).

## Color

Atelier single light skin. 60/30/10 discipline:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--bg` **#F4F1E9** (cream) | App background, **canvas viewport backdrop** (post D-09 re-token) |
| Secondary (30%) | `--panel` **#FCFAF4** / `--panel-2` **#FFFFFF** / `--border` **#E4DECF** | Header bar, Refine rail, cards, bottom action bar, HUD glass, dividers |
| Accent (10%) | `--accent` **#0E6E5C** (green) | See reserved list below |
| Dark accent | `--ink` **#1B1A17** | The single deliberate dark element: the top-bar **Save** pill only |
| Caution (semantic) | `--warn` **#A85A11** (amber) | Low-zoom warning badge only |

**Accent (`--accent` green) reserved for — explicit list (never "all interactive elements"):**
1. Primary advance CTA — the fixed bottom **Next** button (`.btn-chunk`).
2. **Active** view-mode switcher segment (`bg-accent text-on-accent`) — replaces the legacy
   `bg-sky-500 text-white` under D-09.
3. Active/selected `SizeCard` and active `SegmentedControl` segment (Refine).
4. `Slider` filled track + thumb (edge-cleanup, color-count).
5. Active step marker in `StepBar`.
6. The determinate **recompute / matching** progress bar fill (`bg-accent`) — replaces legacy
   `bg-indigo-500` under D-09.

**Destructive color:** none required. This phase introduces **no user-facing destructive action** (the
Step1/2/4 deletions are code-level; removing the manual Recompute button is a simplification). The amber
`--warn` token is caution-only (low-zoom badge) and is **not** a destructive/error color.

**D-09 re-token map (legacy scale name → semantic token) — mandatory for edited surfaces:**

| Legacy (remap shim) | Re-token to | Where |
|---------------------|-------------|-------|
| `bg-slate-950` (viewport) | `bg-bg` | `CanvasWorkspace.tsx` viewport surface |
| `bg-slate-950/80` (loading overlay) | `bg-bg/80` | loading overlay |
| `border-slate-800`, `bg-slate-800` | `border-border`, `bg-border` | canvas border, HUD dividers, progress track |
| `text-slate-300 / -400 / -500` | `text-ink` / `text-muted` / `text-faint` | HUD, hints, captions |
| `bg-sky-500 text-white` (active seg) | `bg-accent text-on-accent` | view-mode switcher active state |
| `bg-indigo-500` (progress fill) | `bg-accent` | loading / recompute progress bar |

> The remap shim currently makes these render "correctly" already; the requirement is to stop **depending**
> on the shim (D-01 removes the theme remnants that motivated it). Visual result is unchanged-or-better;
> the checker validates by class name, not just rendered color.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA (fixed bottom, advance) | **Next** (Back / Next footer, now in the fixed bottom zone — D-05) |
| Top-bar primary pill | **Save** (unchanged) |
| Auto-advance (D-08) | No copy — a successful upload silently advances Upload → Refine (respecting the "Refine reachable only with an uploaded image" gate); the visible step change IS the feedback. |
| Recompute pending state (D-02) | Canvas overlay: indeterminate phase reads **"Recomputing…"**; once the worker ticks it flips to the existing **"Matching colors: {progress}%"**. Reuses the current two-phase loading-overlay component — no new surface. |
| View-mode switcher labels | **Grid Colors** · **Grid + Symbols** · **Original Photo** (unchanged) |
| Canvas hint pill | **drag to pan · scroll to zoom · {N} drills** (unchanged) |
| Low-zoom badge | **⚠️ Low Zoom** (unchanged; caution, not error) |
| Empty state (no image) | Owned by `UploadScreen`; `CanvasWorkspace` renders nothing in the canvas slot when there is no image/match (D-10, unchanged). No new empty-state copy this phase. |
| Error state | Existing frame-scope match/action error banners (hoisted in Phase 23) — surface on any step. No new error copy this phase. |
| Destructive confirmation | **None.** No user-facing destructive action is introduced. The removed "Recompute match" button + amber stale CTA are simply gone (auto-fire replaces them); no confirm dialog. |

## Phase-Specific Interaction & Layout Contracts

The load-bearing content of this phase. Each maps to a CONTEXT decision and a live target; the checker and
executor validate against these, not against a new visual system.

| ID | Contract | Behavior spec | Target |
|----|----------|---------------|--------|
| **D-05** | Fixed 3-zone wizard shell (desktop) | `AtelierShell` root → **Zone 1: fixed top step-bar** (`shrink-0`, `no-print`) → **Zone 2: `flex-1 min-h-0` internally-scrolling content** (step panels + single-mount canvas) → **Zone 3: fixed bottom action bar** (Back/Next, `shrink-0`). In a normal browser window the **Next CTA is always reachable without page scroll**; long Supplies/drill lists scroll **inside** Zone 2. Desktop-scoped so it composes with — not fights — the Phase 24 mobile container-query reflow + sticky-canvas pane. **Single-mount canvas (P20 D-14) is re-zoned, never remounted.** | `AtelierShell.tsx`, `App.tsx:1804` footer → Zone 3 |
| **D-06** | Rail fits browser, canvas is focus | Refine controls rail width-capped (`max-w-[320px]`); canvas viewport is `flex-1`; layout fits browser width at ≥1024px with no horizontal scroll and no rail encroachment. | `RefineScreen.tsx` |
| **D-07** | View-switcher bottom-snap | The Grid / Grid+Symbols / Original switcher **docks to the bottom of the viewport** (inside/above Zone 3), horizontally centered, and **never overlaps or obstructs the canvas**. Keep ≥8px clear of the canvas hint pill (stack or merge; no overlap). Moves from the current top `viewport-hud` position. | `CanvasWorkspace.tsx:85–113` |
| **D-04** | Fit-to-container default zoom, no jump on resize | Persistent **`isFitMode`** flag in `viewer.ts` (behavior-only extension — public zoom API + `onZoomChange` signatures unchanged, P24 D-05 precedent). **Re-fit on:** init, new image load, grid-dimension change, container resize. **Leave fit only on** explicit user zoom in/out or pinch. A SizeCard / custom-size change **re-fits cleanly** — it must **not** preserve the prior scale or jump the zoom. | `viewer.ts`, `CanvasWorkspace.tsx` `onFit` |
| **D-02** | Fully automatic recompute | Remove the manual "Recompute match" button, the amber stale CTA, and the `StepBar` amber stale marker; retire `RefineScreen` `stale`/`onRecompute` props. **SizeCard click → recompute immediately.** **Custom-size input → debounced ~500ms** (400–600ms band, planner's call) after last keystroke, **clamp-guarded** so a half-typed value (e.g. "1") never fires a garbage run. Canvas shows the **"Recomputing…" pending state** (see copy contract) while the worker runs. Reuses the existing soft-invalidate plumbing (auto-fired, not ripped out). | `App.tsx:507–532`, `RefineScreen.tsx`, `StepBar.tsx` |
| **D-08** | Auto-advance on upload | A successful image ingest advances Upload → Refine (step 2), respecting the existing validation gate (Refine reachable only with an uploaded image). No no-op dead end. | `App.tsx` ingest handler |
| **D-03** | Print canvas grid from every step | Canvas `<main>` gets `print:block` on **all four** steps (not just step 2 — `App.tsx:1619`) + a print rule fitting the raster to the page (`max-width:100%`, riding on the D-04 fit default so the printed raster already shows the whole grid). Plain **Ctrl+P produces the canvas grid from anywhere.** The dedicated Supply-Report / legend print paths (`print-only-report-mode`, `print-only-legend-mode`) stay **intact and independent** — no double-print conflict. | `App.tsx:1619`, `src/index.css` `@media print` |
| **D-09** | Re-token dark viewport + zoom buttons | Apply the D-09 re-token map (Color section) to the viewport backdrop, zoom HUD, view switcher, canvas border, and loading overlay; keep zoom buttons at 44px min touch targets. Folds into the D-01 theme-remnant sweep. | `CanvasWorkspace.tsx:81,85–160,184,234` |

**Preserved untouched (guardrail #11 / D-01):** `Step3Canvas.tsx` + its 3 fulfillment handlers + the coupled
`slate-*` fulfillment modals (curated canvas-links modal, checkout modal) and `flags.ts`. Phase 26 owns
their deletion. The re-token/theme sweep **must not** touch the fulfillment-coupled `slate-*` modals.
