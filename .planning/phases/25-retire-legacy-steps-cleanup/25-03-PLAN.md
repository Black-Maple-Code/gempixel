<objective>
Add a persistent `isFitMode` boolean to `CanvasViewer` (`engine/viewer.ts`) so the canvas can
distinguish "resting in fit-to-container" from "user has explicitly zoomed" (D-04). This is a
behavior-only extension of the existing zoom/fit machinery — no public signature changes (matches
the Phase 24 D-05 pinch precedent and the Phase 22 engine-freeze). Expose a read-only accessor so
App (Plans 04/05) can re-fit on dimension/resize changes ONLY while the user is still in fit mode.

Purpose: The viewer half of D-04 (fit is the default zoom; leave fit only on explicit user zoom)
— the foundation for SC8 (no zoom-jump on SizeCard change; fit-to-container default).
Output: `viewer.ts` with `isFitMode` set true in `fitToContainer`, false in `handleZoom`, plus a
public getter and unit tests for the transitions.
</objective>
