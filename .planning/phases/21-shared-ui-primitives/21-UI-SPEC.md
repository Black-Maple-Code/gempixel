# Phase 21 — UI Design Contract

> Visual and interaction contract for the five hand-built Atelier primitives (SegmentedControl, Slider, SizeCard, Pill, Button) that live in `src/ui/` and compose the four v4.0 screens. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope note:** Phase 21 ships the *reusable control layer only* — pure/props-only presentational primitives. It does NOT wire them into Upload/Refine/Supplies/Order (Phase 23), does NOT touch `engine/*` (Phase 22), and does NOT add mobile/touch behavior (Phase 24). All values below are pre-populated from upstream artifacts; no fields were left to guess.

## Design System

| Property | Value |
|----------|-------|
| Tool | **none** (hand-built; shadcn deliberately NOT initialized) |
| Preset | not applicable |
| Component library | **none** — browser-native elements + Tailwind v4 utilities only |
| Icon library | none — inline Unicode glyphs (`✓`, `→`) + pure-CSS marks (`.gem-logo`); no icon package |
| Font (display/titles) | `Newsreader Variable` (self-hosted `@fontsource`), serif, weights 600–700 |
| Font (body/UI) | `Archivo Variable` (self-hosted), sans-serif |
| Font (data/labels/numbers) | `JetBrains Mono` (self-hosted), monospace |

**Why `Tool: none` (shadcn gate resolution):** the stack is Preact/Vite, which normally triggers the shadcn init offer. Initialization is **suppressed by locked upstream decisions**, not by omission:
- CONTEXT.md **D-03** — "Zero new dependencies — hard constraint (SC3). No UI kit, no headless-component lib, no slider lib, no `clsx`/`cva`/`tailwind-merge`." Enforced by a `package.json` diff check at verification.
- REQUIREMENTS.md **Out of Scope** — "Heavy UI / slider / font-loader / PDF libraries" excluded; browser-native + hand-built primitives only.
- ROADMAP.md **SC3** — "built from browser-native elements + Tailwind only — no new UI/slider/util dependency enters `package.json`."

Registry safety gate is therefore **not applicable** (no registry, no third-party blocks).

**Token source of truth:** `src/index.css` — the Atelier `:root` token layer + Tailwind v4 `@theme inline` block. Primitives **consume** existing semantic utilities (`bg-panel`, `bg-panel-2`, `text-ink`, `text-muted`, `text-faint`, `bg-accent`, `text-on-accent`, `border-border`, `text-warn`) and radius/shadow tokens; they **define no new tokens**.

**Styling API (CONTEXT.md D-02):** each primitive exposes a `variant` prop keyed into a local `Record<variant, string>` of Tailwind class strings; a ~5-line hand-rolled `cn(...classes)` join helper lives in `src/ui/` (no `clsx`/`classnames`). Every primitive accepts an optional `className` merged **last** (consumer override wins) and spreads `...rest` onto the underlying native element. `variant` values are string-literal unions, not enums. Reference pattern for prop-interface style, token usage, and a11y wiring: `src/features/wizard/StepBar.tsx`.

## Spacing Scale

8px-based rhythm (design handoff §Spacing). Primitives use Tailwind's 4px-step utilities.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon/label gaps inside a chip, tight inline padding |
| sm | 8px | Default control gap, compact element spacing |
| md | 12px | Button/segment horizontal padding (`px-3`, StepBar precedent) |
| lg | 16px | Between stacked rail controls |
| xl | 24px | SizeCard internal padding (card content block) |
| 2xl | 32px | Section breaks within a rail |

Exceptions (documented, from the handoff — do not treat as violations):
- **6px** (`gap-1.5`, `py-1.5`) — established StepBar precedent for chip gaps and control vertical padding; the new primitives match it for visual consistency.
- **Card padding 22–28px** and **control gaps up to 14px** appear in the handoff prose. Primitives round to the nearest Tailwind step (`p-6` = 24px for SizeCard, `gap-2`/`gap-3` for control rows); the exact 22/28 figures are card-composition concerns for Phase 23, not primitive-level tokens.

## Typography

Primitives are controls, so the roles below are control-scoped. Two core weights (**400 regular**, **600 semibold**); **700 bold** is a documented exception for the current-step label (StepBar precedent), not a general weight.

| Role | Size | Weight | Line Height | Family |
|------|------|--------|-------------|--------|
| Micro-label (mono, uppercase) | 11px | 500 | 1.4 | JetBrains Mono, tracking 0.08–0.14em, `text-faint` |
| Body / control label | 14px | 400 | 1.5 | Archivo |
| Emphasis / button label | 14px | 600 | 1.5 | Archivo |
| Card / section title | 16px | 600 | 1.2 | Newsreader (`.font-display`) |

Notes:
- Mono micro-labels (`COLOR COUNT`, `DRILLS`, `BAGS`, size-card dimension captions) use weight **500** per the handoff — the one deviation from the 400/600 pair, scoped to mono uppercase meta only.
- SizeCard numeric drill count renders in **JetBrains Mono** (data figure), 14px.
- Numbers/codes/counts anywhere on a primitive are mono; prose labels are Archivo; titles are Newsreader.

## Color

60 / 30 / 10 split (design handoff §Colors; tokens in `src/index.css`).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#F4F1E9` (`--bg`) | App/page background behind every primitive |
| Secondary (30%) | `#FCFAF4` (`--panel`) / `#FFFFFF` (`--panel-2`) | Card/rail/control surfaces, input mounts, unselected SizeCard/segment fills |
| Accent (10%) | `#0E6E5C` (`--accent`) | See reserved-for list below |
| Accent tint | `#EAF2EF` | Selected SizeCard fill, selected-segment / "ok" chip background (paired with the green border/text, never used alone as accent) |
| Caution | `#A85A11` (`--warn`) | Reserved for stale/out-of-date markers (StepBar precedent); not used by the 5 new primitives unless a consumer opts in |
| Destructive | none | No destructive action exists in this phase |

**Accent (`#0E6E5C`) reserved for — explicit list:**
- Button `primary` background (`bg-accent` / `text-on-accent`).
- SegmentedControl **selected** segment (green fill or green text + tint per handoff).
- Slider active track fill + handle.
- SizeCard **selected** state: 2px green border + `#EAF2EF` fill.
- Pill `accent`/`ok` variant (green text on `#EAF2EF` tint).
- Key figures a consumer marks as primary (e.g. a size-card drill count when selected) — Phase 23 decides; the primitive only exposes the `selected` styling.

Accent is **never** applied to: all-interactive-elements blanket styling, body text, borders of unselected controls, or the dark `save` Button (that is `#1B1A17` ink, a separate variant — see below).

The **dark "Save" pill** treatment (`#1B1A17` bg / `#F4F1E9` text, `--radius-pill`) is a Button *variant*, not an accent use — it is the top-bar Save affordance from the handoff, encoded as `Button variant="save"`.

Radii (existing tokens, no new ones): cards `--radius-card` 12px · controls/inputs `--radius-control` 8px · pills `--radius-pill` 20px · small chips 4–6px. Card shadow `--shadow-card`.

## Component Inventory

The heart of this phase. Each primitive is **pure/props-only** (App.tsx owns all state — CONTEXT.md D-04); interactive ones are **fully controlled** (`value`/`selected` + `onChange`/`onSelect`, no internal state). Concrete `variant` names are the planner's call against the handoff (CONTEXT.md discretion); recommendations below.

| Primitive | Native element | Key props | Variants (recommended) | A11y contract |
|-----------|----------------|-----------|------------------------|---------------|
| **Button** | `<button type="button">` | `variant`, `onClick`, `className`, `...rest`, children (label) | `primary` (green CTA), `save` (dark pill), `ghost` (bordered secondary — EDIT / "Print packing slip") | Native button semantics; visible focus ring; disabled via native `disabled`; label is consumer-supplied children (no hard-coded copy) |
| **Pill** | `<span>` (or `<button>` if interactive) | `variant`, `className`, children | `neutral`, `accent`/`ok` (green on `#EAF2EF`), `tag` ("BEST" / "BEST FOR ART") | Decorative by default (`aria-hidden` not required if it carries visible text); if used as a toggle, expose `aria-pressed` |
| **SegmentedControl** | container `<div>` + option `<button>`s | `value`, `onChange`, `options[]`, `label` (group) | none (single visual style) | `role="radiogroup"` + accessible group label; options `role="radio"` + `aria-checked`; roving tabindex; ←/→ arrow-key selection; segments must read as clearly mutually-exclusive (no "both look selected" ambiguity) |
| **Slider** | native `<input type="range">` | `value`, `onChange`, `min`, `max`, `step`, `aria-label`, `aria-valuetext` | none | Native range = keyboard operable for free; must set `aria-label` + `aria-valuetext`; accent-styled track/handle via Tailwind (not a custom-drawn div) |
| **SizeCard** | `<button>` (selectable card) | `label`, `gridDims` (e.g. "80×53"), `inches` (formatted string), `drillCount` (number), `selected` (bool), `onSelect`, optional `tag` slot | `selected` vs default (styling, not a `variant` prop) | Selectable via `aria-pressed` or `role="radio"` within a card group; **imports no engine module** — receives fully-computed display values as props (CONTEXT.md D-05) |

Cross-cutting requirements:
- **StepNav is NOT built** — `src/features/wizard/StepBar.tsx` already fulfills it (CONTEXT.md D-01) and stays in `features/wizard/`.
- Optional shared base for Button/Pill/SegmentedControl is allowed only if it cleanly reduces duplication (CONTEXT.md discretion) — not required.
- Retrofitting AtelierShell's existing inline Save pill to consume the new Button/Pill is **discretionary**; do not churn working chrome.
- Tests co-located in `src/ui/__tests__/`; render tests assert a11y roles (`radiogroup`/`radio`/native `range`) and that `variant` + `className` classes apply. Full Vitest suite stays green (SC4; ~255 tests after Phase 20).

## Copywriting Contract

Primitives are label-agnostic containers — screen copy lands in **Phase 23**. The contract here is the *default/example labels and a11y strings* the primitives must support.

| Element | Copy |
|---------|------|
| Primary CTA | **No hard-coded copy** — `Button` renders consumer-supplied children; Phase 23 supplies verb+noun labels ("Browse files", "Match colors →", "Continue to order →", "Place order · $57.00") |
| Empty state heading | Not applicable — primitives are stateless controls with no data-driven empty view (empty/recent-project states belong to Phase 23 screens) |
| Empty state body | Not applicable — see above |
| Error state | Not applicable at primitive level — validation + error banners are owned by App.tsx (Phase 20 pattern); primitives only expose accessible labels so screen-level errors can reference them |
| Destructive confirmation | None — this phase introduces no destructive action |

Accessibility strings the primitives DO own (must be present):
- **SegmentedControl** — a group label (e.g. `aria-label="Edge cleanup"`), and each option's visible text doubles as its radio label.
- **Slider** — `aria-label` (e.g. "Color count") + `aria-valuetext` (e.g. "24 of 26 matched") so the value is announced meaningfully, not just a bare number.
