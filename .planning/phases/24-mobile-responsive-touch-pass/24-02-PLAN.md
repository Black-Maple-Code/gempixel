<objective>
Add hand-rolled multi-touch pinch-to-zoom (plus continued single-finger pan) to the canvas `CanvasViewer` by extending its existing Pointer Events model, set `touch-action: none` on the canvas so the page never scrolls under the gesture, and make the on-screen zoom buttons touch-friendly. Delivers MOBILE-02.

Purpose: On touch devices the user must pinch-zoom and pan the chart with the page held still, aided by the existing on-screen zoom buttons. This is browser-native and additive — no gesture library (CLAUDE.md rejects `panzoom`/Hammer/etc.), no `src/engine/*` public-signature change (the public zoom API + `onZoomChange` are unchanged; only the internal handlers gain an `activePointers` map + pinch branch). Desktop mouse is unregressed (a mouse never yields two concurrent pressed pointers) and the jsdom suite stays green (the pinch branch is gated on two live pointers).
Output: `viewer.ts` multi-touch extension + canvas `touch-action`, jsdom-safe pinch tests in `viewer.test.ts`, and touch-target-sized zoom buttons in `CanvasWorkspace.tsx`.
</objective>
