# Plan 06-01: Build Local Storage Database and Project Switcher

Goal: Implement a split-key local storage database registry and portfolio switcher sidebar panel to manage multiple commission layouts.

## Proposed Changes

### Component Area: Workspace Database & Registry

#### [MODIFY] [App.tsx](file:///C:/Users/rickf/.gemini/antigravity/scratch/gempixel/src/App.tsx)
- Define `ProjectSummary` and `ProjectData` interfaces.
- Add state variables:
  - `projectsRegistry`: `ProjectSummary[]` (initialized from `gempixel_workspace_registry` local storage)
  - `currentProjectId`: `string | null`
  - `showSaveModal`: `boolean`
  - `newProjectName`: `string`
- Add functions:
  - `handleSaveProject(name: string)`:
    - Generates a UUID/timestamp ID if `currentProjectId` is null.
    - Captures active grid matching counts, dimensions, styles, calculator costs, and exclusions.
    - Creates a downsized low-resolution thumbnail data URL of the canvas element.
    - Updates `gempixel_workspace_registry` and stores project details under `gempixel_project_${id}`.
  - `handleLoadProject(id: string)`:
    - Fetches details from `gempixel_project_${id}`.
    - Restores column, row, aspect ratio, exclusions, drill style, base kit, and pricing calculator configurations.
    - Triggers color match recalculation using the restored grid array to paint the canvas instantly.
  - `handleDeleteProject(id: string, event: Event)`:
    - Removes project details from local storage and deletes entry in registry.
    - Resets workspace config to default if deleted project was currently active.
  - `handleCreateNewProject()`:
    - Clears active canvas image, resets calculator pricing, sets columns/rows to default (80x53), and clears active exclusions.
- Render the "My Commissions" collapsible details list at the top of the Left Sidebar panel.
- Render a backdrop modal dialog for naming and saving the current workspace layout.
