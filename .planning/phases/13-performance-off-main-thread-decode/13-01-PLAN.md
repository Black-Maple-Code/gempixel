<objective>
Relocate the image resample + `getImageData` readback + `boxSampleImage` call OFF the main thread INTO the existing `matcher.worker.ts`, behind a single new message contract that transfers a zero-copy `ImageBitmap` (D-01/D-04) â€” NOT a raw-Blob decode inside the worker (D-03 rejected that for parity risk + Firefox `createImageBitmap(blob)`-in-worker issues). This is a PURE relocation â€” no output-affecting change to the box-sample algorithm, the `maxDimension=2000` cap, or the match loop.

Purpose: Today `useDiamondArtMatch.ts:112-113` runs `getImagePixels()` (a `drawImage` resample to 2000px + full-frame `getImageData`) and `boxSampleImage()` (a millions-of-pixels averaging loop) SYNCHRONOUSLY on the main thread before invoking the already-off-thread matcher, janking the UI on every match for large images. Moving them into the worker satisfies PERF-01.
Output: The worker/client/hook message contract changed atomically (all three callers of `match()` updated together so `tsc` stays green), the worker gains an injectable decode preamble, and `worker.test.ts` routed through the decode seam with all four prior behaviors preserved.

This plan is the atomic contract change. Plan 13-02 consumes the new `loadingPhase` signal for the overlay copy; Plan 13-03 is the manual parity/UX gate.
</objective>
