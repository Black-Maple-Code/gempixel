## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| localStorage/project-import -> App state | A restored `ProjectData` (or hand-crafted/tampered blob) supplies `selectedVendor`, which drives canvas-cost money math. |
| App state -> calculateCanvasCost | An out-of-union vendor value could reach the pricing engine via a type-unsafe restore. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-15-01 | Tampering | calculateCanvasCost unknown-vendor branch | high | mitigate | Return `null` (never `0.0`) for an unknown vendor; App.tsx skips `setCanvasBaseCost` on null, so a tampered/legacy vendor can never yield a free $0 canvas. |
| T-15-02 | Tampering | Load-time selectedVendor restore | medium | mitigate | `normalizeVendor` remaps any non-{lumaprints,finerworks} value to lumaprints at read time; migration is read-only and non-destructive (save re-persists the normalized value), so a corrupt vendor cannot corrupt the rest of the project. |
| T-15-SC | Tampering | npm/pip/cargo installs | low | accept | No new packages introduced this plan; nothing to audit. |
</threat_model>
